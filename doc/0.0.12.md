# v0.0.12 — 用户系统 + Auth（Email 注册/登录 + Session）

## 背景

Sprint 1 第二阶段：在 v0.0.11 的 PostgreSQL/Prisma 地基上，引入完整的用户认证系统。
前端加入路由（react-router-dom）和全局状态（zustand），实现 Login / Register / Onboarding / Profile 四个页面。

不影响现有功能（搜索 / 分词 / TTS / i18n / 单词本全部继续工作）。

---

## 一、技术选型

| 需求 | 方案 | 原因 |
|------|------|------|
| 密码加密 | `bcryptjs` | 标准，cost=12 |
| 会话令牌 | `jsonwebtoken` (30 天 JWT) | httpOnly cookie 存储 |
| Cookie 解析 | `cookie-parser` | Express 中间件 |
| 前端路由 | `react-router-dom` v7 | 多页面 SPA |
| 全局状态 | `zustand` | 轻量，无 Provider 样板 |

**Google OAuth**：按钮已渲染（占位），v0.0.13 接入。

## 三、认证流程

### 注册
```
POST /auth/register { email, password, name }
→ 创建 users 行 + student_core 行（事务）
→ 签 JWT → 写 httpOnly cookie
→ 返回 { user }
```

### 登录
```
POST /auth/signin { email, password }
→ 验证 bcrypt 哈希
→ 签 JWT → 写 httpOnly cookie
→ 返回 { user }
```

### Session 检查
```
GET /auth/session
→ 读 cookie → 验 JWT
→ 返回 { user } 或 { user: null }
```

### Cookie 配置
- `httpOnly: true`（XSS 防护）
- `sameSite: "lax"`
- `secure: true`（仅生产环境）
- 有效期：30 天
