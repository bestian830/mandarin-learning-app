# v0.0.4 — "多音字修复 + 拼音布局优化"

## 功能

修复多音字拼音错误，改进逐字拼音的视觉布局。

| 功能 | 说明 |
|------|------|
| 多音字消歧 | 替换 pinyin-pro 为 pinyin v4 + nodejieba，按词库分词后确定正确读音 |
| 拼音布局 | 每个汉字下方单独显示对应拼音，间距合理，不再聚成一行 |
| 中文直接输入修复 | 用户输入中文时，底部原文朗读按钮正常工作（修复 zh-Hans 语言代码映射） |

---

## 问题根因

### 多音字错误（如"转行"→错误显示 zhuǎn xíng）

- **根因**：`pinyin-pro` 默认按单字查音，不考虑词语上下文
- **现象**：转行（职业改变）中的"行"被读为 xíng（行走），而非正确的 háng（行业）
- **修复**：引入 `nodejieba` 分词，先将"转行"识别为一个词，再查词典得到 háng

### 中文输入原文 TTS 无声音

- **根因**：Azure Translator 检测中文返回 `"zh-Hans"`，前端 `toLangTag()` 映射表无此 key，回退为 `en-US`
- **修复**：`toLangTag()` 对所有以 `zh` 开头的代码统一映射为 `zh-CN`

### 拼音布局混乱

- **根因**：列宽由最宽元素（拼音文字）决定，tracking-wide 使拼音过宽，字间距看起来不均匀
- **修复**：每列设定 `min-width: 1.6rem`，移除 `tracking-wide`，使用 `whitespace-nowrap`

---

## 技术实现

### pinyin v4 + nodejieba

```typescript
import pkg from "pinyin";
const { pinyin, STYLE_TONE } = pkg;

// nodejieba 分词后按词查音，正确处理多音字
pinyin("转行", { style: STYLE_TONE, segment: "nodejieba" });
// => [["zhuǎn"], ["háng"]] ✓
```

对比旧方案（pinyin-pro，单字模式）：
```
转行 → zhuǎn xíng  ✗
银行 → yín xíng    ✗
音乐 → yīn lè      ✗
```

新方案（pinyin + nodejieba，词库分词）：
```
转行 → zhuǎn háng  ✓
银行 → yín háng    ✓
音乐 → yīn yuè     ✓
```
