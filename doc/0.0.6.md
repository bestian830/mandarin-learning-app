# v0.0.6 — 词频统计数据库（SQLite）

## 功能

每次搜索后自动将分词结果写入本地 SQLite 数据库，相同词语累计计数，
供日后统计高频/低频词及制作教学数据使用。

| 功能 | 说明 |
|------|------|
| 自动写入 | 每次 `/api/search` 返回后，分词结果异步写入 `data/words.db` |
| 计数累加 | 同一词重复出现时 count +1，更新 last_seen |
| 标点过滤 | `punct` / `other` 分类不写入，只记录有意义的词语 |
| 零影响 | fire-and-forget（`setImmediate`），不阻塞搜索响应速度 |

## 部署架构

```
云服务器（Azure/GCP 等）
├── mandarin_master 后端
└── data/words.db  ← 实时写入

M910Q（家用服务器）
└── words.db       ← 定期 rsync 同步，用于本地分析
```

同步命令示例：
```bash
rsync user@cloud-server:/app/data/words.db ~/words_backup.db
```

## 数据库结构

```sql
CREATE TABLE word_stats (
  word        TEXT PRIMARY KEY,   -- 简体词语
  traditional TEXT,               -- 繁体（与简体不同时存储）
  pinyin      TEXT,               -- 拼音
  category    TEXT,               -- 词性分类
  count       INTEGER DEFAULT 1,  -- 出现次数
  first_seen  TEXT,               -- 首次查询时间
  last_seen   TEXT                -- 最后查询时间
);
```

---

## 常用查询

```bash
# 高频词 Top 20
sqlite3 data/words.db "SELECT word, pinyin, count FROM word_stats ORDER BY count DESC LIMIT 20"

# 按词性查动词
sqlite3 data/words.db "SELECT word, count FROM word_stats WHERE category='verb' ORDER BY count DESC"

# 只出现一次的生词
sqlite3 data/words.db "SELECT word, pinyin FROM word_stats WHERE count=1"

# 导出 CSV
sqlite3 -csv -header data/words.db "SELECT * FROM word_stats ORDER BY count DESC" > words.csv
```
