# v0.0.10 — 多卡组单词本 + 中文词典释义

## 功能概述

本版本完成三个方向的升级：
1. **单词释义语言改为中文** — 用台湾教育部词典（moedict）替代 Azure Translator，让用户以中文学中文
2. **管理员拼音修正工具** — 支持在线编辑拼音并持久化到数据库
3. **多卡组单词本** — 类似 Anki，可创建多个命名卡组分类管理词汇

---

## 一、中文词义：moedict API 集成

### 动机

原始实现调用 Azure Translator 将中文词语翻译成用户母语（英文、日文等）显示在单词本和搜索浮窗中。这与"以中文学中文"的教学理念相悖，且翻译成本较高。

### 方案

使用台湾教育部「萌典」公开 API（`moedict.tw`），免费、无需 Key，返回繁体中文定义，再用 opencc-js 转为简体显示。

```
GET https://www.moedict.tw/a/{Traditional}.json
→ data.h[0].d[0].f  （释义字符串，去除 ` 和 ~ 格式符后使用）
```

### 新增文件

**`backend/src/lib/dictionary.ts`**
- 内存 session 缓存（进程内复用，重启清空）
- 5 秒超时，失败静默返回空字符串
- `stripFormatting(text)` 去除 moedict 格式符（`` ` `` 和 `~`）
- 调用 `toTraditional()` 将搜索词转繁体后查询，结果用 `toSimplified()` 转回简体

**`backend/src/lib/traditional.ts`**（新增函数）
- `toSimplified(traditional: string): string` — opencc tw→cn 转换

### 接口变化

`POST /api/translate-word` — 参数保持兼容（`word`, `toLang`），但 `toLang` 已不再使用，统一返回简体中文释义。

### 兜底逻辑

- 词未收录：返回空字符串，前端不显示释义（不显示错误）
- 网络超时 / API 异常：同上，静默处理

## 二、管理员拼音修正

### 架构

数据库已有 `word_stats.pinyin_override` 列（上个版本添加）。本版本补全：
- 后端写入接口加鉴权
- 前端编辑 UI
- 词库面板和搜索结果读取修正值

### 激活方式

访问 `/?admin=<ADMIN_SECRET>` → token 存入 `localStorage["mm_admin_token"]`，刷新后保持。
`/?admin=logout` 清除 token。

### 后端

**`backend/src/routes/vocab.ts`**
- `requireAdmin` 中间件：读取 `Authorization: Bearer <token>` 头，与 `process.env.ADMIN_SECRET` 比较
- `PATCH /api/word/:word/pinyin`（需鉴权）：更新 `pinyin_override` 列
- `GET /api/words/overrides`（公开）：返回所有已修正词的列表

**`backend/src/db/index.ts`** — SQL 查询加 COALESCE
```sql
-- 改前（allStmt / topStmt）
SELECT pinyin FROM word_stats ...
-- 改后
SELECT COALESCE(pinyin_override, pinyin) AS pinyin FROM word_stats ...
```
这样词库面板（VocabPanel）刷新后也显示修正后的拼音。

### 前端

- `Home.tsx` — VocabRow 组件：管理员模式下拼音可点击进入编辑框，Enter/失焦触发 PATCH；失败时边框变红 1.5 秒
- `WordBook.tsx` — 打开时 fetch `/api/words/overrides`，调用 `fc.applyPinyinOverrides()` 将修正写入 localStorage 闪卡

### `lib/flashcards.ts` 新增

```typescript
export function applyPinyinOverrides(overrides: Record<string, string>): void
// 遍历所有卡组，有变化时写回 localStorage
```

## 三、多卡组单词本

### 存储设计

| Key | 内容 |
|-----|------|
| `mm_decks` | `Deck[]` JSON，所有卡组元数据 |
| `mm_fc_{deckId}` | `FlashCard[]` JSON，每个卡组的闪卡 |
| `mm_active_deck` | string，当前活跃卡组 id |
| `mm_flashcards` | 旧数据 key（迁移后删除）|

### 自动迁移

首次加载时，若 `mm_flashcards` 存在：
1. 创建名为 "My Words" 的默认卡组
2. 将旧数据写入 `mm_fc_{newId}`
3. 删除 `mm_flashcards`

无旧数据则跳过。

### 新增类型

**`frontend/src/types/index.ts`**
```typescript
export interface Deck {
  id: string;
  name: string;
  createdAt: string;
}
```

### flashcards.ts API 重构

所有卡片函数新增 `deckId` 参数；新增卡组管理函数：

| 函数 | 说明 |
|------|------|
| `getDecks()` | 返回所有卡组 |
| `createDeck(name)` | 新建卡组 |
| `deleteDeck(id)` | 删除卡组及其闪卡 |
| `renameDeck(id, name)` | 重命名 |
| `getActiveDeckId()` | 当前活跃卡组（有 fallback）|
| `setActiveDeckId(id)` | 切换活跃卡组 |
| `exists(word)` | 跨所有卡组检查 |
| `applyPinyinOverrides(map)` | 遍历所有卡组应用修正 |

### WordBook UI

两个视图通过 `currentDeckId: string | null` 切换：

**卡组列表视图**（`null`）
- 显示所有卡组：名称 + 词数
- 底部输入框新建卡组（Enter 确认）
- hover 显示删除按钮

**卡组详情视图**（`string`）
- 顶部 ← 返回按钮 + 卡组名
- Start Review 始终显示（有词即可），使用全部词（不按 due 过滤）
- 词列表：简体字+拼音逐字对应、繁体右对齐、中文释义
- 底部手动输入添加词

### 搜索页卡组选择器

`Result.tsx` 中点击 + 按钮时：
- 只有 1 个卡组 → 直接添加
- 多个卡组 → tooltip 内变为卡组选择列表，点击即添加到对应卡组

### i18n 新增字段

| Key | 英文默认值 |
|-----|-----------|
| `wbNewDeckPlaceholder` | `"New deck name..."` |
| `wbChooseDeck` | `"Add to deck"` |

同时在 `backend/src/routes/i18n.ts` 的 `SOURCE_EN` 中补充，触发 Azure 翻译；前端缓存版本升至 **v7**。

## 四、其他修复

### Review 去除 due 计数

单词本的定位是记录用户收藏的词，复习频率由用户自己决定，不应强制 SM-2 调度。
- 卡组列表和卡组详情头部移除"X due"显示，只显示词数
- Start Review 按钮始终传入全部卡片（`cards`），不再筛选到期卡

SM-2 算法的 box 更新逻辑保留在后台（FlashcardReview 自评时仍会更新），但不对用户展示。

### 发音按钮不触发翻卡

`FlashcardReview.tsx` 中，卡片正面的 SpeakerButton 外层 div 加 `onClick={e => e.stopPropagation()}`，防止点击发音时事件冒泡到 `handleFlip`。
