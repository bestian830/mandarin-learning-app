# v0.0.11 — PostgreSQL + Prisma 数据库地基（盈利改造 Sprint 1·第一波）

## 背景

从 v0.0.11 起进入 **盈利路线 9 周 MVP 改造**（完整计划：`~/.claude/plans/velvet-kindling-steele.md`）。
本版本完成 Sprint 1 第一阶段：**引入 PostgreSQL + Prisma ORM**，为用户系统、AI 同学陪聊、
充值余额等功能铺设完整的数据库地基。

不影响现有功能（搜索 / 分词 / 拼音 / 单词本 / TTS / i18n / 拼音修正全部继续工作）。

## 一、技术选型

**Prisma 6（而非原生 pg）**

| 优势 | 说明 |
|------|------|
| 自动类型生成 | 17 张表自动生成 100% 类型安全的 TypeScript Client |
| 迁移管理 | `prisma migrate dev` 自动 diff + 生成 migration SQL，上线改表结构安全 |
| Auth.js 兼容 | `@auth/prisma-adapter` 直接接，无需手写 adapter |
| 可读性 | `schema.prisma` 比 SQL DDL 更简洁，关系一眼看清 |

**Prisma 版本**：6.19.3（最新稳定版，使用新的 `prisma.config.ts` 配置方式）

### 学生档案 4 层设计

注入 AI 的总 token 约 1400，**恒定不随使用次数增长**：

```
Layer 1 student_core            ~500 token  永远注入 prompt
Layer 2 student_rolling_summary ~300 token  每 10 节课重写
Layer 3 student_recent_lessons  ~600 token  最近 3 条（滚动覆盖）
Layer 4 lesson_archive          --------   不注入，仅用户查询
```

### 关键设计点
- UUID 主键：`@id @default(uuid())`（Prisma 自动处理 UUID 生成）
- `transcript`（完整对话）：`Json` 类型，映射 PostgreSQL JSONB
- `interestTags` / `personalityTags` 等数组：`String[]`
- `Decimal` 精度：余额/金额用 `@db.Decimal(10, 2)` 确保财务计算准确
- AI 只写文本字段（`aiSummary`、`summaryText`、`shortSummary`），不写结构化数据
- `student_recent_lessons` 在应用层维护 "只保留最近 3 条" 的滚动逻辑

| 镜像 | `postgres:16-alpine` |
| 数据卷 | `mandarin_master_pgdata` |

### 常用命令

```bash
cd backend
docker compose up -d        # 启动
docker compose down         # 停止（保留数据）
docker compose down -v      # 停止并清空数据
npm run db:migrate          # 新建 migration（改了 schema.prisma 后运行）
npm run db:studio           # Prisma Studio 可视化管理界面
```

## 四、Prisma Client 封装

### `backend/src/db/client.ts`

单例 `PrismaClient`，防止 dev 热重载时创建多个连接。

```typescript
import { prisma } from '../db/client.js'

// 查询示例
const user = await prisma.user.findUnique({ where: { email } });
const decks = await prisma.userDeck.findMany({ where: { userId } });
```

**生成路径**：`src/generated/prisma/`（由 `schema.prisma` 自动生成，`.gitignore` 已排除）

**注意**：改完 `schema.prisma` 后必须运行 `npx prisma generate` 更新 client，或者 `npm run db:migrate`（migrate 时自动 generate）。
