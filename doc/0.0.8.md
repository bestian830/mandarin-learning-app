# v0.0.8 — 分词修复：HMM 误合并 + 未登录词系统化处理

## 问题

nodejieba 使用 HMM（隐马尔可夫模型）处理未登录词时，会在特定上下文将多个已知汉字错误合并为一个 `tag="x"` 的 token，导致这些字没有 tooltip 显示：

| 输入 | jieba 错误输出 | 正确应为 |
|------|---------------|---------|
| 今晚哪支队伍**会赢**？ | `"会赢"(x)` 整体 | `"会"(v)` + `"赢"(v)` |
| 他的能力**很强** | `"很强"(x)` 整体 | `"很"(d)` + `"强"(a)` |
| 会议室在**三楼** | `"三楼"(x)` 整体 | `"三"(m)` + `"楼"(n)` |

注：词典内的复合词（会议、能力、会议室、会考、可能性…）标签为 `n/v` 等，**不受影响**。

---

## 根本原因

ICTCLAS `"x"` 标签有两种含义：
1. **标点符号**（如 `？`、`，`）→ 不显示 tooltip（正确）
2. **未登录词 / HMM 误合并**（如 `"会赢"`）→ 之前也被当成标点，tooltip 缺失（错误）

---

## 修复方案

### 核心逻辑

对 `tag="x"` 且含汉字的 token，逐字拆开并对每个汉字单独调用 `tag(char)` 重新获取词性：

```
"会赢"(x) → tag("会") → v → verb（动词）
             tag("赢") → v → verb（动词）

"很强"(x) → tag("很") → d → adverb（副词）
             tag("强") → a → adj（形容词）
```

### 实现

`backend/src/lib/segment.ts` 重构为三层函数：

- `buildSegment()` — 构建单个 Segment 对象（抽取重复逻辑）
- `tokenToSegments()` — 单个 jieba token → 一个或多个 Segment，处理三种情况：
  1. 标点 / "x" 不含汉字 → 单个 punct segment
  2. "x" 含汉字 → 逐字拆开重新 tag
  3. 正常 token → 直接映射
- `segmentText()` — `tagged.map()` 改为 `tagged.flatMap()`，允许一个 token 展开

`frontend/src/components/Result.tsx` 新增 `other` 类到 `CATEGORY_META`：
```typescript
other: { en: "Word", zh: "词", color: "#6b7280" },
```

## 验证

```

→ "会"(动词) + "赢"(动词) 各有独立 tooltip ✓

搜索：他的能力很强
→ "很"(副词) + "强"(形容词) 各有独立 tooltip ✓

搜索：会议室在三楼
→ "会议室"(名词) 整词 tooltip + "三"(数量词) + "楼"(名词) ✓

搜索：这次会考你准备好了吗
→ "会考"(名词) 保持整词，不被拆分 ✓
```

## 未来维护说明

此方案无需手动维护词典。任何被 jieba HMM 误合并的汉字组合都会自动逐字处理，每个字从 jieba 主词典获取正确词性。唯一例外是：词典外的单字本身也无法识别时，降级显示为灰色 `"Word / 词"` 标签。
