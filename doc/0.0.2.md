# v0.0.2 — "Voice"

## 功能

在 v0.0.1 翻译结果卡片基础上新增语音朗读功能，参考 Google Translate 交互设计：

| 位置 | 功能 |
|------|------|
| 中文结果区 | 喇叭按钮 + 速度选择器（½× / 1× / 1½×） |
| 英文原文区 | 喇叭按钮（固定正常速度） |

**交互细节**
- 点击喇叭开始朗读，再次点击停止
- 播放中：按钮变红、扩散光环动画、显示双音波图标
- 速度选择器：三档分段式按钮（½× / 1× / 1½×），选中高亮红色
- 中英混合文本（如翻译结果含英文专有名词）：自动按语言段拆分，中文段用中文语音、英文段用英文语音，两种语言都能正确朗读
- 语音选择器：中文喇叭旁 `▾` 下拉列出系统所有中文语音，点选即生效，选择结果通过 `localStorage` 持久化（key: `mm_zh_voice_uri`），下次打开自动恢复

## 技术栈

### 使用方案
| 方案 | 说明 |
|------|------|
| **Web Speech API** (`SpeechSynthesis`) | 浏览器内置，完全免费，无需 API key，无调用限制 |

### 语音质量优化
- 自动从系统语音列表中优先选取高质量中文语音（Tingting / Sinji / Google 普通话 / Microsoft Xiaoxiao 等）
- 英文语音优先选取 Samantha / Google US English / Microsoft Aria 等
- 通过 `onvoiceschanged` 事件异步加载完整语音列表

### 混合语言处理
```
"Python是一门programming language"
  ↓ splitByLang()
["Python", "是一门", "programming language"]
  ↓ 分别用 en-US / zh-CN / en-US 语音引擎
  ↓ 依次放入 SpeechSynthesis 队列
```

## 已知限制

- Web Speech API 语音质量依赖系统安装的语音包，macOS / iOS 效果最佳
- Windows / Android 上内置中文语音质量参差不齐
- Chrome 在标签页失去焦点时偶发停止朗读（浏览器限制，非 bug）
- 如需生产级中文语音质量，可升级至 Azure Cognitive Services TTS（免费 50 万字/月）
